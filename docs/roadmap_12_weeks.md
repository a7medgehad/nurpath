# 12-Week Execution Roadmap

## Week 1 - Foundation

- Publish repo with CI and templates.
- Enable Discussions Q&A.
- Create issue labels for data, retrieval, safety, frontend, and eval.

## Week 2-3 - Data Pipeline

- Ingest verified public-domain/open-licensed Arabic-Islamic sources.
- Normalize source and passage metadata.
- Track provenance in a machine-readable manifest.

## Week 4-5 - RAG Core

- Add embedding indexing into Qdrant.
- Add lexical fallback and rerank layer.
- Enforce citation-span validation before answer release.

## Week 6-7 - Agentic Tutor

- Extend intent taxonomy.
- Improve opinion extraction and conflict summarization.
- Add confidence calibration and abstention policy tests.

## Week 8 - Learning UX

- Add objective progression graph UI.
- Add quiz generation and mastery dashboards.
- Add Arabic pedagogy UX refinements.

## Week 9 - Evaluation

- Expand bilingual benchmark set.
- Track faithfulness, abstention quality, and ikhtilaf fidelity.

## Week 10 - Community Release

- Publish demo + transparency report.
- Open first good-first-issue batch.

## Week 11-12 - Hardening

- Optimize latency and retrieval quality.
- Close high-priority bugs.
- Prepare first stable release tag.
