# Contributing

## Development Standards

- Open an issue before large features.
- Keep PRs focused and tied to acceptance criteria.
- Include tests for behavior changes.
- Keep citations and safety constraints intact.

## Commit Style

Use conventional commits where possible.

## Review Expectations

- Require at least one review for non-docs changes.
- Keep merge policy consistent and quality-focused.

See `README.md` and `docs/architecture.md`.
